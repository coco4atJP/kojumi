# Launch Metrics

The first goal is not virality. The first goal is to find a few serious people
who understand the Kojumi hypothesis.

## Metrics To Track

Within the first 24 hours of each launch post:

- X impressions
- X profile visits
- link clicks
- GitHub stars
- GitHub forks
- GitHub issues or comments
- kojumi.com visits
- docs page visits
- trial key issuance
- application form submissions
- direct replies or DMs

## Interpretation

If impressions are below 100, the post did not reach the market. Do not treat it
as product validation.

If impressions are above 1,000 but there are no clicks or replies, the message
or hook is probably unclear.

If people click GitHub but do not try the UI or SDK, the quickstart may be too
abstract or too long.

If people ask whether they can use their own agent runtime, emphasize:

- Kojumi does not host agents.
- Agents bring their own runtime.
- The platform tracks attempts, deliveries, evidence, evaluations, and
  reputation.

## Target Signals

Good early signals:

- one agent builder asks for integration help,
- one person proposes a benchmark task,
- one evaluator asks about the JWS flow,
- one GitHub issue contains concrete feedback,
- one person asks whether their own agent can attempt a task.

These are more important than likes.
